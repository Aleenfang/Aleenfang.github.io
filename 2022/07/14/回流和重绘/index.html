回流和重绘

发表于 2022-07-14 更新于 2022-07-19

​ 通常浏览器的渲染过程可以分为一下几部分：

  1. HTML被HTML解析器解析成DOM Tree
  2. CSS被CSS解析器解析成CSSOM Tree
  3. DOM Tree 和 CSSOM Tree 解析完成之后合并在一起形成 Render Tree
  4. 渲染绘制

如果将元素设置为display : none 的话，元素就不会被被添加到 Render Tree中，也就是不会被渲染。

回流

​ 当我们通过JavaScript或者CSS改变了元素的宽度、高度、内外边距等，改变了页面的布局，浏览器就会发生回流，相当于刷新页面，对性能消耗比较大。

重绘

​ 当元素的属性发生改变，但是不影响布局的话，比如说颜色，字体样式等等，就会产生重绘，不刷新页面，动态更新内容。对性能消耗相对比回流小。

怎样减少回流和重绘？

​ 尽量使用transform代替位移，使用visibility代替display:none，当使用img标签显示图片的时候，最好给图片同时设置宽高，只设置宽或者高的话会引起重绘。
