微信小程序--事件监听

发表于 2022-08-03 更新于 2022-08-05

微信绑定事件的方式

bindeventName 或者 bind:eventName

比如 bindTap 就是监听点击事件

监听事件的时候会传入一个回调函数，回调函数有一个参数：event

event 的属性：

  • type 事件类型
  • timeStamp 页面打开到触发事件所经过的毫秒数
  • target 触发事件的组件的一些属性值的集合 触发事件的源头组件，事件冒泡，
  • currentTarget 当前组件的一些属性值的集合 当前事件所绑定的组件 ， 参考事件冒泡
  • detail 额外的信息
  • changedTouches 触摸事件，当前变化的触摸点信息的数组

事件传参

在绑定事件的标签上面 设置 data - *

在vue中父组件给子组件传值数据丢失问题中

发表于 2022-07-30

错误场景

​ 在vue中，父组件给子组件传的值需要异步获取，在浏览器刷新之后，这可能会造成子组件报错 can not read data not defined ，子组件报没拿到父组件传过来的值，但是我拿vue develop tools 查看子组件，却可以看到从父组件传过来的值，子组件却没有将值渲染出来。

错误原因

​ 当浏览器刷新之后，整个vue实例就会被重新加载，父组件就会调用API 重新发送网络请求，但是这个时候问题来了，网络请求是异步的，当子组件已经开始渲染的时候父组件还没有拿到数据，所以子组件就会报错。

​ 还有一种情况，如果数据的对象比较大，可能浅层的数据已经可以拿到了，但是深层的数据还是undefined。

​ 所以我的解决办法就是给子组件加上一个 v-if 条件就是需要传给子组件的数据，因为浅层数据拿到的时候深层数据不一定拿到，所以最好是将深层数据作为条件。

transform中rotate和translate混用的问题

发表于 2022-07-30

transform可以在浏览器不用reLayout的情况下移动元素的位置，对动画性能比较友好。

​ 其中，translate是使元素平移，rotate是使元素旋转

​ 在旋转的时候，会将元素的 x， y轴跟着一起旋转，所以当面临混用的情况，就可以将translate放在前面，rotate放在后面，这样就会先平移再旋转，不会影响到translate

遍历数组的两种常用方法

发表于 2022-07-30

1. Array.prototype.forEach( callback(item,index,array ) )

​ forEach传入一个回调函数，回调函数有3个参数

  1. 当前遍历的value
  2. 当前value的下标
  3. 当前遍历的array

2. Array.prototype.map(callback(item,index,array ) )

​ map参数跟forEach一样

map 和 forEach的区别：

map 会根据回调函数的返回值返回一个新的数组，如果回调函数返回true，就会把当前遍历的value添加到新数组中去，返回false就不添加，而forEach函数没有返回值
