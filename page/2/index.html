原型和原型链

发表于 2022-07-13 更新于 2022-07-14

什么是构造函数？

​ 构造函数就是一个普通的函数，区别是它跟普通函数的调用方式不同，普通函数在函数名后面加上一对小括号”( )”调用，构造函数通过new操作符调用并可以通过小括号”( )”给构造函数传入参数。按照惯例，一般普通函数用小写字母开头，构造函数用大写字母开头。

​ 构造函数里面的this指向的是实例对象，所以给this设置属性实际上是设置到了实例对象上，并没有设置到原型对象上。

​ 在JavaScript中创建一个函数的时候，会同时为这个函数创建一个prototype属性，这个属性指向构造函数的原型对象。所有原型对象自动获得一个名为constructor的属性，指向与之关联的构造函数。

调用构造函数会发生什么？

​ 调用构造函数会产生一个新的实例对象。这个实例对象内部的[[prototype]]属性会指向构造函数的原型对象。这个原型对象也就称为实例对象的原型。

​ 通常我们可以通过 _proto_（浏览器提供） 属性或者Object.getPrototypeOf ( ) （JavaScript提供）方法来获取实例对象的原型。

​ 应该注意的是，通过实例对象可以找到它的原型，但是通过原型是找不到它的实例对象的。同一个构造函数构建的实例对象都共享同一个原型对象。

原型链

​ 实例对象内部的[[prototype]]属性指向原型对象，原型对象通过内部的[[prototype]]属性指向Object对象，Object对象内部的[[prototype]]指向null。原型链说的就是对象间的这种链接关系。

​ 通常可以用isPropotyOf ( )来判断一个对象是不是另一个对象的原型。

​ 当每次访问实例对象上的属性或方法时，那么引擎首先会在当前对象查找，如果找不到就会自动沿着原型链自底向上查找。如果找到了之后就直接返回，不会继续再向上查找了，找不到就会返回undefined。

事件循环

发表于 2022-07-08 更新于 2022-07-15

JavaScript是一门单线程异步的脚本语言，也就是只有一个调用栈，同一时刻只能执行一条语句。所谓异步，就是这个任务可能会耗费很长的事件，比如网络请求、文件IO等等……让整个程序都阻塞在这个地方，大大影响了程序的性能。为了解决这个问题，引入了异步的概念。就是先执行其他不容易导致阻塞的部分，其他部分执行完了之后再执行异步部分。

什么是异步？

​ JavaScript执行上下文执行到异步代码的部分的时候并不会立刻执行这个里面的任务，而是会先将这个代码块放入一个任务队列当中，暂时搁置，先去执行别的同步代码，当全部同步代码执行完毕之后，再按照异步任务放入任务队列的顺序挨个执行。

怎么实现代码异步执行？

​ 最简单的异步当属定时器(setTimeout)了，当我们在定时器中传入回调函数之后，回调函数总是会在程序的同步代码和promise之后执行。

​ 其实将异步任务继续细分的话，可以分为宏任务和微任务。宏任务一般是由宿主环境定义的，微任务是由JavaScript自己定义的。每个宏任务都会先执行，然后执行它附属的微任务，必须等微任务执行完毕之后才能开始执行下一个宏任务。

宏任务

​ 主要的宏任务声明途径有：

  1. 新程序或者子程序的执行
  2. 事件的回调函数
  3. setTimeout() setInterval()

微任务

​ 主要的微任务声明途径有：

  1. Promise.then() Promise.catch() Promise.finally()
  2. MutationObserver

事件循环

​ 事件循环描述的是JavaScript执行代码的顺序，有三大组成要素

  1. 调用栈
  2. 宏任务队列（消息队列）
  3. 微任务队列

执行顺序

  1. 执行JavaScript语句
  2. 碰到函数就将函数压入调用栈
  3. 碰到微任务和宏任务声明就将它们压入相应的队列
  4. 调用栈清空
  5. 执行微任务队列
  6. 微任务队列清空
  7. 浏览器渲染
  8. 执行宏任务队列
  9. 返回到1
  10. 宏任务队列清空

数组扁平化

发表于 2022-07-07

什么是数组扁平化？

​ 对于一个数组，它里面的元素可能也是一个数组，也就是说这个数组是嵌套的，或者说这个数组是一个多维数组。而数组扁平化则是将这个嵌套的数组解套，也就是将这个多维数组变为一维数组。

数组扁平化的具体操作

  1. 在Array对象的原型上增加数组扁平化的方法，方便使用
  2. Array.prototype.map()遍历当前数组，如果是数组就递归调用数组扁平化方法，不是就返回包装的数组，map()方法得到的结果是一个二维数组
  3. 使用Array.prototype.concat()方法将map()方法得到的二维数组转化为一维数组并返回。

标准盒子模型和IE盒子模型

发表于 2022-07-06 更新于 2022-07-15

​ 网页中盒子模型分为两种，标准盒子模型（content-box）和IE盒子模型（border-box）。

标准盒子模型和IE盒子模型的区别

  1. 标准盒子模型的border和padding都是在原来设置的宽高的基础上往外扩张的，所以盒子的总的宽高会大于我们一开始设置的宽高，我们一开始设置的宽高始终等于content部分的宽和高。

  2. IE盒子模型的border和padding都是在原来设置的宽高的基础上往内扩张的，所以盒子的总的宽高是我们一开始设置的宽高，

    在设置border和padding之后content的大小会被压缩。

  3. 对于margin属性，标准盒子模型和IE盒子模型都是往外扩张的。也就是不计入盒子最终的大小。

工作者线程

发表于 2022-07-05

工作者线程

​ 通常来说，JavaScript是一门单线程的脚本语言，这意味着JavaScript在同一时刻只能做一件事情。但是web Worker的出现，使得JavaScript可以将工作交给独立的实体来完成，也就有了多线程的特性。

​ 工作者线程其实是浏览器在原始页面环境之外再分配的一个二级子环境，这个子环境不能操作、访问Dom，因为如果主线程和子线程都能同时操作Dom的话就很容易出现问题。

工作者线程的特点：

  1. 工作者线程是以实际线程实现的，也就是说一个工作者线程对应一个浏览器底层的线程
  2. 工作者线程并行执行：虽然页面和工作者线程都是单线程的JavaScript环境，但是每个环境中的指令可以并行执行。
  3. 工作者线程可以共享某些内存：工作者线程可以使用 SharedArrayBuffer 在多个环境间共享内容，并使用Atomics接口实现并发控制，解决并发资源争用问题。
  4. 工作者线程不共享全部内存：除了SharedArrayBuffer 外，从工作者线程进出数据需要复制或转移
  5. 工作者线程和可能不在同一个进程里
  6. 创建工作者进程的开销比较大：工作者线程启动的成本比较高，占用内存大,所以尽量不要有频繁的启动关闭操作

工作者线程的类型：

   1. 专用工作者线程：只能被创建它的页面使用
   2. 共享工作者线程：可以被多个不同的上下文使用，任何与创建共享工作者同源的脚本都可以使用共享工作者线程
   3. 服务工作者线程：服务工作者线程负责拦截、重定向、修改页面发出的请求等功能。

主线程与工作者线程之间的数据传输

  1. 结构化克隆算法
    ​ 结构化克隆算法可用于两个独立的上下文之间传输数据，该算法由浏览器后台实现。使用postMessage() 传递数据就是默认使用的这种方法。在传递对象时，浏览器会遍历该对象，并在工作者线程的上下文中生成它的一个副本。结构化克隆算法支持除了Symbol之外的所有原始数据类型。
注意事项：

  1. 复制之后，副本跟原数据之间不会有任何联系，也就是说这是个深拷贝。

  2. 不能克隆 Error对象，Function对象，DOM节点

  3. 对象的属性描述选项，get，set方法不会被复制

  4. 深拷贝会比较消耗性能，所以不适合传输复制过大的数据

  1. 可转移对象
    使用可转移对象可以把所有权从一个上下文转移到另一个上下文。在不方便复制大量数据的情况下，可以使用这个功能。
注意事项
    1. 当数据从一个上下文转移到另一个上下文之后，原来的上下文就会丢失这个数据的引用
    2. 支持的数据类型有    ArrayBuffer  Messageport  ImageBitmap OffscreenCanvas
    3. 可以在其他对象中嵌套可转移对象，在发送数据的时候，可转移对象会转移，而其他的数据会复制
3. SharedArrayBuffer

​ SharedArrayBuffer 是一种特殊的ArrayBuffer 它能作为一个公共内存被不同的上下文使用，并使用Atomics API 解决资源争用问题。但是由于安全问题，SharedArrayBuffer 的使用比较麻烦。

​ SharedArrayBuffer被postMessage() 传输的时候 既不是复制也不是转移，postMessage 会将它的引用传给 其他的线程

nextTick的用法

发表于 2022-07-05 更新于 2022-07-06

​ nextTick是等待下一次DOM更新的工具方法。因为Vue视图是异步更新的，Vue会监听数据变化，在同一事件循环中所有的数据变化完成之后再执行视图更新。所以当我们对数据进行修改之后，可能是拿不到更新之后的DOM的，这个时候就需要借助nextTick工具方法来确保我们拿到的是更新之后的DOM。

nextTick使用场景

  1. 在created生命周期进行dom操作的时候（比如说使用echarts时需要绑定dom）
  2. 在数据变化的时候，需要使用数据变化之后的DOM

script标签

发表于 2022-07-05

将JavaScript引入HTML的方式主要是使用script标签。script标签可以放在>标签里面，放在这里除非使用async、defer属性，不然都无法访问到dom对象，因为执行它里面的代码的时候dom元素还没有挂载，也可以放在标签里面，除非使用async、defer属性，不然放在这里只可以拿到script标签前面的dom元素，拿不到script标签后面的dom元素，因为执行的时候dom元素还没有挂载。如果不给script标签添加async属性或者defer属性，浏览器会同步下载JavaScript代码并同步执行，这个时候会阻塞浏览器的其他动作。

script标签的属性

  1. async

    表示应该立即下载脚本，但是异步执行脚本里面的内容。也就是说不会阻碍HTML其他的正常动作执行。只对外部引入脚本有效。当有多个async时，浏览器不能保证是否按照先后顺序执行。

  2. defer

    浏览器对这个属性的script标签的处理方式类似，区别是：浏览器会按照标签出现的先后次序执行脚本。

  3. crosorigin

    用于配置相关请求CORS(跨域资源共享)设置。添加crosorigin之后，控制台会显示一个比较详细的报错信息。

  4. src

    用户引入外部的js文件，src属性的值是一个url，当浏览器解析到这个地方的时候，会向该url发送一个get请求，这个get请求不受同源策略的限制，但是下载过来的js代码再发送网络请求的时候又会受到同源策略的限制。由script标签的get请求不受同源策略限制的特性，衍生出了JSONP这种解决跨域问题的方法。当为script标签设置了src属性之后，浏览器就会只执行外部引入的JavaScript，忽略script标签内部的行内JavaScript。

  5. type

    当将type的值设置为module时，浏览器就会将该script标签视为一个模块，从而可以使用es6的import语法导入模块和export语法导出模块。

节流函数(throttle)的实现

节流函数：就是当一个函数可能重复调用的频率比较高的时候，我们规定一个时间间隔，函数的执行时间间隔低于我们规定的时间。

举例：

​ 首先定义一个节流函数，跟防抖函数一样，我们的节流函数也是一个高阶函数。

​ 为了解决被执行函数的arguments和this丢失的问题，使用apply来绑定this和传入参数

​ 由于在函数在指定时间间隔内只能执行一次，所以我们加入计时器

​ 当调用这个函数的时候，如果timer已经存在，说明函数刚执行过，而且时间间隔还没到，所以直接返回，程序不会执行到下面调用被执行函数这一行。

​ 在这里我并没有选择把被执行函数放在定时器里面，因为放在里面的话被执行函数就不能在第一时间执行，必须等待一个时间周期之后才能开始执行，而放在外面的话函数就能在第一时间执行。

​ 当指定的时间间隔到了之后，我们就清除timer，这样程序就不会执行if里面的语句直接返回了，而是可以继续往下面走。当被调用函数被执行了之后，我们再一次调用定时器给timer赋值，这样在指定时间间隔内程序就又会直接返回，不会继续往下走。

​

​ 到这里，我们的节流函数就实现了。
