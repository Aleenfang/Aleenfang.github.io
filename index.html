css中带 @ 的规则

发表于 2022-07-18

​ 在CSS中，可以将规则分为两种：带@的规则和普通规则。带@的规则都不需要选择器，而普通规则则需要配合选择器使用。

​ 常用的带@的规则有

  • @charset
  • @import
  • @media
  • @key-frames
  • @font-face

@charset

​ @charset时用于提示CSS文件使用的字符编码方式，必须出现在CSS文件的最前面，这个规则不影响页面上的展示效果。

@import

​ @import用于引入一个外部CSS文件，除了@charset规则不会被引入，@import可以引入另外一个CSS文件的全部内容。

@media

​ @media就是媒体查询使用的规则，它能够对设备的类型进行判断。可以根据不同的设备匹配不同的样式规则。

@key-frames

​ @key-frames用于定义动画的关键帧，通常配合animation规则使用

@font-face

​ @font-face用于定义一种字体，阿里巴巴矢量图标库的icon-font就是利用这个特性实现的

MVC和MVVM的关系

发表于 2022-07-15

​ mvc是服务端的概念，mvvｍ是客户端的概念，他们不能相提并论。

MVC

​ mvc是后端逻辑分层的概念，可以分为m（model），v（view）,c（controller）三大部分。

view

​ view就是视图层的意思，也就是前端页面，视图层 （v）和 控制器（c）通过 http 协议 进行通信，由view请求controler提供的的api接口（request），然后控制器（c）收到请求（request）之后，经过路由（router）将request分发

controler

​ controler就是控制器的意思，通常在基于node.js的接口开发里面，有路由分发模块（router）、 中间件模块（middleware）、控制器模块（controler），这三个模块都属于controler层。从 view 发送到后端的 请求 （request）会先到达入口文件（app.js）,入口文件收到请求之后会调用路由模块（router）进行路由分发,经过路由分发之后,请求会经过中间件(middleware) 进行相应的处理,处理之后会到达controler模块,controler模块会根据请求 调用 数据模型模块（model）,结合model里面的数据并进行逻辑处理之后就会将数据 以http response 的方式返回给 视图层（view）

model

​ model就是数据模型的意思，它是职能单一的，只负责数据库的操作，执行sql语句，进行数据的增、删、改、查 。 数据模型里面对增删改查等操作进行了封装，并对外部暴露了封装好之后的方法，使得controler在对数据库操作的时候不需要每次都写sql语句。同时，model也负责连接数据库。

MVVＭ

​ mｖｖｍ是前端的逻辑分层概念。可以分为三层：每个页面保存的数据模型（ｍ）数据调度者（ｍｖ）视图层（ｖ）。

model

​ 当用户打开一个页面的时候，页面会向服务器发送请求，并将数据保存在本地。这些数据就是当前页面的数据模型。数据模型要显示到页面上的话就要将数据给到ｍｖ。在ｖｕｅ中，响应式数据功能实现了ｍｏｄｅｌ和ｍｏｄｅｌｖｉｅｗ的数据双向绑定

modelview

​ 当数据从服务器下载到本地之后，数据就会流向ｍｖ模型，然后由ｍｖ模型决定什么时候将数据渲染到视图层。当ｍｖ拿到来自于页面的数据之后，也会将数据保存到数据模型中去。在ｖｕｅ中，框架所做的事情就相当于这里的数据调度者。ｖｕｅ框架实现了与ｖｉｅｗ之间的数据双向绑定

view

​ 视图层负责将来自于ｍｖ的数据显示到页面上来。当用户触发页面监听的事件，输入数据之后，将数据返回给数据调度模型（ｍｖ）。

localStorage和sessionstorage的区别

发表于 2022-07-14

Web Storage出现的目的是为了解决通过客户端存储不需要频繁发送回服务器的数据时使用Cookie的问题。

sessionStorage

sessionStorage是跨会话的存储机制，这意味着数据只会存储到浏览器关闭。存储在sessionStorage中的数据不受刷新的影响，可以在浏览器崩溃重启之后恢复。

localStorage

localStorage是永久存储机制，也就是说当浏览器关闭了之后，存储的数据依然在。要访问同一个localStorage对象，页面必须来自同一个域（子域不可以）、相同端口上使用相同的协议。

​ 浏览器给Web Storage使用的空间大小给出了限制，一般会限制每个源5MB。

css选择器和优先级

发表于 2022-07-14

常用的CSS选择器有

  1. 元素选择器 （权重 1）
  2. 群组选择器 （ h2 , p {} 右边花括号里面的规则应用到左边两个选择器上）
  3. 通用选择器 （权重0 * { } ）
  4. 类选择器 （权重10 .class-name { } ）
  5. ID选择器 （权重100 #id { } ）
  6. 后代选择器 （h2 p { } ）
  7. 子元素选择器 （h2 > p { }）
  8. 兄弟选择器 ( h2 + p { } 意思是p元素紧跟在h2元素后面，且同属于一个父元素)
  9. 后续兄弟选择器 ( h2 ~ p { } 意思是同一父元素下所有在h2元素后面的p元素)
  10. 伪类选择器
  11. 伪元素选择器

​ 当有多条规则作用于同一个元素是，渲染引擎就会计算不同规则的权重，由权重最高的那一条规则生效。

css权重的计算就是每出现一个相应权重的选择器，就将它的权值加入总权重当中来。

​ 当元素脱离正常的文档流时，元素之间就会互相堆叠，可以通过设置z-index属性来控制堆叠的顺序。

回流和重绘

发表于 2022-07-14

​ 通常浏览器的渲染过程可以分为一下几部分：

  1. HTML被HTML解析器解析成DOM Tree
  2. CSS被CSS解析器解析成CSSOM Tree
  3. DOM Tree 和 CSSOM Tree 解析完成之后合并在一起形成 Render Tree
  4. 渲染绘制

如果将元素设置为display : none 的话，元素就不会被被添加到 Render Tree中，也就是不会被渲染。

回流

​ 当我们通过JavaScript或者CSS改变了元素的宽度、高度、内外边距等，改变了页面的布局，浏览器就会发生回流，相当于刷新页面，对性能消耗比较大。

重绘

​ 当元素的属性发生改变，但是不影响布局的话，比如说颜色，字体样式等等，就会产生重绘，不刷新页面，动态更新内容。对性能消耗相对比回流小。

怎样减少回流和重绘？

​ 尽量使用transform代替位移，使用visibility代替display:none等。

原型和原型链

发表于 2022-07-13 更新于 2022-07-14

什么是构造函数？

​ 构造函数就是一个普通的函数，区别是它跟普通函数的调用方式不同，普通函数在函数名后面加上一对小括号”( )”调用，构造函数通过new操作符调用并可以通过小括号”( )”给构造函数传入参数。按照惯例，一般普通函数用小写字母开头，构造函数用大写字母开头。

​ 构造函数里面的this指向的是实例对象，所以给this设置属性实际上是设置到了实例对象上，并没有设置到原型对象上。

​ 在JavaScript中创建一个函数的时候，会同时为这个函数创建一个prototype属性，这个属性指向构造函数的原型对象。所有原型对象自动获得一个名为constructor的属性，指向与之关联的构造函数。

调用构造函数会发生什么？

​ 调用构造函数会产生一个新的实例对象。这个实例对象内部的[[prototype]]属性会指向构造函数的原型对象。这个原型对象也就称为实例对象的原型。

​ 通常我们可以通过 _proto_（浏览器提供） 属性或者Object.getPrototypeOf ( ) （JavaScript提供）方法来获取实例对象的原型。

​ 应该注意的是，通过实例对象可以找到它的原型，但是通过原型是找不到它的实例对象的。同一个构造函数构建的实例对象都共享同一个原型对象。

原型链

​ 实例对象内部的[[prototype]]属性指向原型对象，原型对象通过内部的[[prototype]]属性指向Object对象，Object对象内部的[[prototype]]指向null。原型链说的就是对象间的这种链接关系。

​ 通常可以用isPropotyOf ( )来判断一个对象是不是另一个对象的原型。

​ 当每次访问实例对象上的属性或方法时，那么引擎首先会在当前对象查找，如果找不到就会自动沿着原型链自底向上查找。如果找到了之后就直接返回，不会继续再向上查找了，找不到就会返回undefined。

事件循环

发表于 2022-07-08 更新于 2022-07-15

JavaScript是一门单线程异步的脚本语言，也就是只有一个调用栈，同一时刻只能执行一条语句。所谓异步，就是这个任务可能会耗费很长的事件，比如网络请求、文件IO等等……让整个程序都阻塞在这个地方，大大影响了程序的性能。为了解决这个问题，引入了异步的概念。就是先执行其他不容易导致阻塞的部分，其他部分执行完了之后再执行异步部分。

什么是异步？

​ JavaScript执行上下文执行到异步代码的部分的时候并不会立刻执行这个里面的任务，而是会先将这个代码块放入一个任务队列当中，暂时搁置，先去执行别的同步代码，当全部同步代码执行完毕之后，再按照异步任务放入任务队列的顺序挨个执行。

怎么实现代码异步执行？

​ 最简单的异步当属定时器(setTimeout)了，当我们在定时器中传入回调函数之后，回调函数总是会在程序的同步代码和promise之后执行。

​ 其实将异步任务继续细分的话，可以分为宏任务和微任务。宏任务一般是由宿主环境定义的，微任务是由JavaScript自己定义的。每个宏任务都会先执行，然后执行它附属的微任务，必须等微任务执行完毕之后才能开始执行下一个宏任务。

宏任务

​ 主要的宏任务声明途径有：

  1. 新程序或者子程序的执行
  2. 事件的回调函数
  3. setTimeout() setInterval()

微任务

​ 主要的微任务声明途径有：

  1. Promise.then() Promise.catch() Promise.finally()
  2. MutationObserver

事件循环

​ 事件循环描述的是JavaScript执行代码的顺序，有三大组成要素

  1. 调用栈
  2. 宏任务队列（消息队列）
  3. 微任务队列

执行顺序

  1. 执行JavaScript语句
  2. 碰到函数就将函数压入调用栈
  3. 碰到微任务和宏任务声明就将它们压入相应的队列
  4. 调用栈清空
  5. 执行微任务队列
  6. 微任务队列清空
  7. 浏览器渲染
  8. 执行宏任务队列
  9. 返回到1
  10. 宏任务队列清空

数组扁平化

发表于 2022-07-07

什么是数组扁平化？

​ 对于一个数组，它里面的元素可能也是一个数组，也就是说这个数组是嵌套的，或者说这个数组是一个多维数组。而数组扁平化则是将这个嵌套的数组解套，也就是将这个多维数组变为一维数组。

数组扁平化的具体操作

  1. 在Array对象的原型上增加数组扁平化的方法，方便使用
  2. Array.prototype.map()遍历当前数组，如果是数组就递归调用数组扁平化方法，不是就返回包装的数组，map()方法得到的结果是一个二维数组
  3. 使用Array.prototype.concat()方法将map()方法得到的二维数组转化为一维数组并返回。

标准盒子模型和IE盒子模型

发表于 2022-07-06 更新于 2022-07-15

​ 网页中盒子模型分为两种，标准盒子模型（content-box）和IE盒子模型（border-box）。

标准盒子模型和IE盒子模型的区别

  1. 标准盒子模型的border和padding都是在原来设置的宽高的基础上往外扩张的，所以盒子的总的宽高会大于我们一开始设置的宽高，我们一开始设置的宽高始终等于content部分的宽和高。

  2. IE盒子模型的border和padding都是在原来设置的宽高的基础上往内扩张的，所以盒子的总的宽高是我们一开始设置的宽高，

    在设置border和padding之后content的大小会被压缩。

  3. 对于margin属性，标准盒子模型和IE盒子模型都是往外扩张的。也就是不计入盒子最终的大小。

TypedArray

发表于 2022-07-05

定型数组是一种ArrayBuffer视图。

创建定型数组

​ 创建定型数组的方式包括读取已有缓冲，使用自有缓冲，填充可迭代结构，以及填充基于任意类型的定型数组。除此之外，使用.from() 和.of() 也可以创建定型数组

​ TypedArray 的二进制表示对操作系统而言更加易用，所以JavaScript引擎可以优化算数运算、位运算和其他针对定型数组的常见操作，所以定性数组的速度很快

​ TypedArray的很多方法是跟普通数组相同的，但是由于定型数组不能更改大小，所以没有concat(),pop(),push(),shift(),unshift(),splice()这些方法，因为这些方法会改变原数组的大小。
