在vue中父组件给子组件传值数据丢失问题中

发表于 2022-07-30

错误场景

​ 在vue中，父组件给子组件传的值需要异步获取，在浏览器刷新之后，这可能会造成子组件报错 can not read data not defined ，子组件报没拿到父组件传过来的值，但是我拿vue develop tools 查看子组件，却可以看到从父组件传过来的值，子组件却没有将值渲染出来。

错误原因

​ 当浏览器刷新之后，整个vue实例就会被重新加载，父组件就会调用API 重新发送网络请求，但是这个时候问题来了，网络请求是异步的，当子组件已经开始渲染的时候父组件还没有拿到数据，所以子组件就会报错。

​ 还有一种情况，如果数据的对象比较大，可能浅层的数据已经可以拿到了，但是深层的数据还是undefined。

​ 所以我的解决办法就是给子组件加上一个 v-if 条件就是需要传给子组件的数据，因为浅层数据拿到的时候深层数据不一定拿到，所以最好是将深层数据作为条件。

transform可以在浏览器不用reLayout的情况下移动元素的位置，对动画性能比较友好。

闭包

发表于 2022-07-29 更新于 2022-07-30

闭包（closure）

闭包 = 环境 + 控制

​ JavaScript中，以函数能访问其定义时的环境中的变量的方式得以实现。

​ JavaScript有闭包的特性是因为作用域链的存在，使得内部函数在访问一个未定义的变量的时候，会沿着作用域链向外层查找。而作用域链是单向的，只能由内层往外层找，不能外层找。

​ 就是在控制函数外面套上一层函数，由外层函数提供环境，内层函数提供控制，模拟了私有变量的特性：外部取不到，只能在内部控制函数拿。

​ 应用场景：我们在函数里面经常需要维护一个外部的变量来保存当前执行的状态，比如说做一个分页的时候需要维护一个index，这个这个index的声明不能直接放在函数里面，这样就失去了保存状态的意义。放在全局变量里又会显得代码很杂乱，其他函数也能访问到，这不利于维护，因此，我们可以在函数外层再套一个函数，用于声明环境变量，然后在内层函数里面写相关的逻辑控制。

​

​

正常函数里面的变量会在函数调用完之后销毁，

但是如果使用了闭包，那么函数里面声明的变量就会被带出来，垃圾销毁机制就不能清理这个变量

所以使用闭包有可能会导致内存泄漏的问题

解决方式是在使用完变量之后，就要将变量名赋值为空，没有地方引用这个变量之后，垃圾销毁机制就会清除这个变量

img标签

发表于 2022-07-19

img

​ img标签用于在网页中显示图片，它是替换型元素，给img标签设置src属性之后，浏览器加载HTML的时候先会根据给img标签设置的宽、高排版，占个位置，然后会向src的目标地址发起get请求，在从服务器拿到图片文件之后，这个图片就会替换当前的img标签。

小问题

  • 如果只设置宽或高，就会等比例缩放，如果同时设置宽高的话就可能破坏图片的宽高比。
  • 如果只设置宽或高，就会引起浏览器的重绘，影响到性能。
  • 当通过设置宽高放大图片的话是有损的，会影响画质，当给图片缩小的话又浪费带宽资源，减慢页面加载速度因此最好是根据需要从服务器请求不同尺寸的图片。
  • srcset和sizes属性可以在不同的屏幕大小和特性下使用不同的图片源

picture

​ picture元素可以根据屏幕的条件为其中的img元素提供不同的源，其中media属性跟CSS中的@media是一样的
