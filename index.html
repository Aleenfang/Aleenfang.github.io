React--render函数

发表于 2022-08-21

当使用setState更新数据的时候

  • React会调用当前组件的render函数更新视图。
  • 子组件的render函数也会被调用。
  • 这个时候以当前组件为根的整个组件树都会调用render函数重新比较VNode，并决定是否渲染。

如果子组件依赖的数据并没有发生改变，那么它就不需要调用render函数。类组件为了解决这个问题：

  1. 子组件使用shouldComponentUpdate(nextProps,nextState)生命周期钩子，并通过比较新旧依赖数据返回Boolean类型决定是否需要更新当前组件。
  2. 子组件不再继承Component，改为继承PureComponent，这样React就会自动比较Props和state，决定是否需要更新组件。

PureComponent的问题

  • ​ PureComponent本质上也是对新旧的依赖数据进行比较，但是由于性能消耗考虑，这个比较只是浅层比较，当深层数据改变的时候，即使是组件依赖这个深层数据，但是也不会更新。

如何解决PureComponent不比较深层数据的问题？

​ 在父组件使用setState改变数据的时候，通过直接改变state中数据的引用地址，子组件就能监听到数据已经发生了改变，从而触发更新。

改变数据引用地址的方法：

  • 对于数组，可以通过调用concat，拓展运算符等浅复制的方法，产生一个新的数组，通过将新数组赋给旧数组就可以实现改变数组类型数据的引用地址
  • 对于对象，可以通过调用assign，拓展运算符等浅复制方法，产生一个新的对象，通过将新对象赋值给旧对象，就可以改变对象数据类型的引用地址

那么，函数式组件怎么解决这个问题呢？

  • ​ 使用高阶组件 React.memo()

React.memo()

​ React.memo()是一个高阶组件，传入一个函数式组件作为参数，memo会返回一个新的组件，这个组件就类似于继承PureComponent的类组件，也会浅层监听Props，state，condex的数据变化，决定是否需要重新渲染。

  • 对于深层数据，memo高阶组件任然不能监听到，解决方法还是通过改变数据引用地址使子组件能监听到数据变化。

React--setState

发表于 2022-08-21

setState是用来干什么的？

​ 在React开发中，我们不能直接通过修改state的值进行视图更新。因为我们直接修改state之后，React并不知道我们对State进行了修改。由于React中的数据不是响应式的，所以需要通过setState来手动通知React我们对state进行了修改，从而进行视图更新。

在class组件中，setState是从哪里来的呢？

​ 在class组件中，setState是从Component类上继承过来的。

setState更新到底是同步的还是异步的？

​ 在大多数情况下，setState都是异步的，因为如果每次修改state都重新进行页面渲染的话会比较消耗性能，为了解决这个问题，React会异步更新视图，在同一个事件循环周期内的更新视图任务都会放入一个队列中，然后等待上次事件循环的渲染任务完成之后，将这个队列中的所有更新视图任务都出列，进入新老VNode的比较，尽可能减少对真实dom的操作。

如何获取异步更新之后的值？

  • ​ setState 函数可以传入两个参数，第一个参数可以是函数或者对象，第二个参数是一个回调函数，这个回调函数会在更新之后执行
  • ​ 在componentDidUpdate生命周期函数里面可以获取到更新之后的数据

setState在哪些情况下是同步的？

​ 在setTimeOut中，原生Dom事件中setState是同步的。这里需要注意的是，在JSX中绑定的事件基本上都是React的合成事件，并不是原生的，原生事件需要通过原生方法获取dom对象之后才能绑定。

vue-router 刷新之后params就丢失的问题

发表于 2022-08-07

​ 当在vue-router中使用params传参数的时候，参数不会显示在页面上，但是当再次刷新页面的时候，会导致 params 参数丢失。

解决方案

  • 使用sessionStorage存储参数，在目标页面使用参数的时候判定一下，如果没有就从sessionStorage中拿
  • 使用query，query会将参数整合在url中，不会导致参数丢失

二元逻辑运算符

二元逻辑运算符在平时的项目开发中特别常见，下面是关于他们的一些总结。

1. && 逻辑与

​ 通常在一些条件表达式里比较常见，如果两边都为true的话，表达式整体就返回true，有一个false的话，表达式整体就返回false

2. || 逻辑或

​ 一般也是在条件表达式里比较常见，如果两边有一个为true的话，整体就为true，如果都为false 的话就为false,然后就是在赋值语句的时候，还有一种特殊的用法。

3. ？？ 空值合并预算符

​ 如果 ?? 前面是 null 或 undefined，取后面的默认值

微信小程序--分包

发表于 2022-08-03 更新于 2022-08-05

分包

为了加快小程序的加载速度，一般会将小程序分为多个模块，依次加载。

  1. 主包

    1. 主包包含tabBar页面资源，公共资源

  2. 分包

    1. 分包包含非tabBar的页面，私有资源

加载策略

  1. ​ 在小程序启动时，默认下载主包并启动主包里面的页面，也就是tabBar
  2. 当用户进入分包的某个页面的时候，客户端会把对应的分包下载下来，下载完成之后再展示，也就是按需加载

分包的体积限制

  • 整个小程序的所有分包大小不超过16M
  • 单个主包、分包大小不超过2M

使用规则

  • 主包放在主文件夹的pages文件夹里
  • 分包放在主文件加内的分包文件夹里
  • 在app.json里设置的时候 ， 主包 在pages选项里声明，分包在subpackages里面声明

打包规则

  1. 小程序会按subpackages的配置进行分包，subpackages之外的目录将被打包到主包中

  2. 主包也可以有自己的paegs，也就是 app.json里面最外层的pages

  3. tabBar页面必须在主包内

  4. 分包之间不能嵌套，也就是说分包里不能再有分包，但是可以有很多个页面

微信小程序--component

发表于 2022-08-03 更新于 2022-08-05

微信小程序的组件一般放在component文件夹下面

局部引用

当page需要使用到组件的时候，需要在page的json文件中引入component

全局引用

当组件在很多页面上都会用到的时候，就可以在app.json里面对组件进行全局引用。

组件与页面的区别

  • 组件的 .json文件中，要设置 component:”true”
  • 组件的 .js 文件中，调用的时Component() 函数， 页面 调用的时 Page() 函数
  • 组件的事件处理函数，函数等 都 需要定义在methods中

样式隔离

  • app.wxss 中的全局样式对组件无效
  • 只有class选择器有样式隔离效果，id选择器，属性选择器，标签选择器都不受样式隔离的影响
  • 可以在组件的 .json 文件中 或者 .js 文件中 添加 “styleIsolation” 使组件样式隔离失效
    • isolated default：true 表示启用样式隔离
    • apply-shared default: false 表示关闭组件的样式隔离，页面wxss可以影响组件，组件wxss不影响页面
    • shared default :false 表示页面wxss和组件wxss相互影响

组件传值

  • 属性绑定：子组件通过在传入Component函数的对象中添加 properties 属性，来声明要接受的数据名称，这个跟vue 的props类似，这个也可以有完整定义属性的方式和简化定义属性的方式

  • 事件绑定： 跟vue里面类似，子组件给父组件传值可以通过 子组件触发事件，然后 父组件监听子组件触发的事件实现。

    1. 父组件 定义一个函数，用于接受子组件传过来的值 childrenData(e){}
    2. 子组件 通过触发事件将值传给父组件 this.triggerEvent(‘eventName’,{dataObject})
    3. 父组件 通过监听子组件触发的事件， 在接受函数中 拿到 子组件传过来的值 bind:eventName=”childrenData”
    4. 父组件 通过接收函数的参数 获得 子组件传过来的值 e.detail

  • 获取子组件实例：类似于vue里面的组件ref，

    • 父组件通过this.selectComponent(‘id或class选择器’)获取子组件实例，从而直接访问子组件的任意数据和方法，

data 和 properties的区别

在小程序组件中，data和properties都是可读可写的，读取的时候直接用，写入的时候需要使用 setData（{}）函数

  • data 更倾向于存储私有数据
  • properties更倾向于存储外界传递到组件中的数据

纯数据字段

当data中的字段既不会渲染到界面上，也不会传递给其他组件的时候，就可以将它定义为纯数据字段，或者说这个字段不是响应式的。

优点： 有助于提升组件的性能
使用规则：在Component 的 options节点中，指定pureDataPattern为一个正则表达式，符合这个正则表达式的字段将成为纯数据字段。

组件生命周期

  • created 此时还不能调用setData ， 是能用于给this添加一些自定义属性字段
  • attached 这个时候this.data初始化完毕，可以使用setData了
  • ready
  • moved
  • detached 当组件离开页面节点树后触发，退出一个页面会触发所有子组件的这个生命周期，适合做一些清理工作
  • error

组件所在页面的生命周期

在组件内部，可以获取使用该组件的页面的生命周期的情况

  • show 组件所在页面被展示时执行
  • hide 组件所在页面被隐藏时执行
  • resize 组件所在的页面尺寸变化时执行

插槽

在vue中父组件给子组件传值数据丢失问题中

发表于 2022-07-30

错误场景

​ 在vue中，父组件给子组件传的值需要异步获取，在浏览器刷新之后，这可能会造成子组件报错 can not read data not defined ，子组件报没拿到父组件传过来的值，但是我拿vue develop tools 查看子组件，却可以看到从父组件传过来的值，子组件却没有将值渲染出来。

错误原因

​ 当浏览器刷新之后，整个vue实例就会被重新加载，父组件就会调用API 重新发送网络请求，但是这个时候问题来了，网络请求是异步的，当子组件已经开始渲染的时候父组件还没有拿到数据，所以子组件就会报错。

​ 还有一种情况，如果数据的对象比较大，可能浅层的数据已经可以拿到了，但是深层的数据还是undefined。

​ 所以我的解决办法就是给子组件加上一个 v-if 条件就是需要传给子组件的数据，因为浅层数据拿到的时候深层数据不一定拿到，所以最好是将深层数据作为条件。
