关于性能

发表于 2022-07-19

​ 一个网页的性能直接决定了用户的体验，而性能可以分成很多个方面，最重要的几个点是：

  • 页面加载性能
  • 动画与操作性能
  • 内存、电量消耗
其中，用户感知最明显的是页面加载性能，其次是动画与操作性能。往往用户打开一个网页不愿意等待很久，如果等了一会浏览器还处于“白屏“状态，用户就会索性关掉网页。而动画、操作卡顿也会大大影响用户的使用心情。

基本概念

​ Flex布局支持横向和纵向。我们把Flex延伸的方向称为主轴，与主轴垂直的方向称为交叉轴。主轴开始的位置叫做main start，结束的位置叫做main end。交叉轴开始的位置叫做cross start，交叉轴结束的位置叫做cross end。

​ Flex布局分为两个部分，Flex容器和 Flex-item

容器

​ 容器就是用来装需要进行布局的元素的父元素。

  • ```css
    flex-flow: || /flex-direction和flex-wrap的简写形式/

  • ```css
    align-items: flex-start | flex-end | center | baseline | stretch; /item在交叉轴上的对齐方式/

item的属性

img标签

发表于 2022-07-19

img

​ img标签用于在网页中显示图片，它是替换型元素，给img标签设置src属性之后，浏览器加载HTML的时候先会根据给img标签设置的宽、高排版，占个位置，然后会向src的目标地址发起get请求，在从服务器拿到图片文件之后，这个图片就会替换当前的img标签。

小问题

  • 如果只设置宽或高，就会等比例缩放，如果同时设置宽高的话就可能破坏图片的宽高比。
  • 如果只设置宽或高，就会引起浏览器的重绘，影响到性能。
  • 当通过设置宽高放大图片的话是有损的，会影响画质，当给图片缩小的话又浪费带宽资源，减慢页面加载速度因此最好是根据需要从服务器请求不同尺寸的图片。
  • srcset和sizes属性可以在不同的屏幕大小和特性下使用不同的图片源

picture

​ picture元素可以根据屏幕的条件为其中的img元素提供不同的源，其中media属性跟CSS中的@media是一样的

meta标签

发表于 2022-07-18

​ meta标签时一组键值对，它是一种通用的元信息表示标签。比如描述、针对搜索引擎的关键词以及刷新频率。

​ meta标签是放在head标签里面的，它可以有任意多个。一般的meta标签由name和content两个属性来定义。name表示元信息的key，content表示元信息的value。

​ 常用的meta元素定义的元数据的类型包括以下几种：

  • 如果设置了name属性，meta元素提供的是文档级别的元数据，应用于整个页面
  • 如果设置了 http-equiv 属性，meta元素则是编译指令，提供的信息与类似命名的HTTP头部相同
  • 如果设置了charset属性，meta元素是一个字符集声明，告诉文档哪种字符编码。

css中带 @ 的规则

发表于 2022-07-18

​ 在CSS中，可以将规则分为两种：带@的规则和普通规则。带@的规则都不需要选择器，而普通规则则需要配合选择器使用。

​ 常用的带@的规则有

  • @charset
  • @import
  • @media
  • @key-frames
  • @font-face

@charset

​ @charset时用于提示CSS文件使用的字符编码方式，必须出现在CSS文件的最前面，这个规则不影响页面上的展示效果。

@import

​ @import用于引入一个外部CSS文件，除了@charset规则不会被引入，@import可以引入另外一个CSS文件的全部内容。

@media

​ @media就是媒体查询使用的规则，它能够对设备的类型进行判断。可以根据不同的设备匹配不同的样式规则。

@key-frames

​ @key-frames用于定义动画的关键帧，通常配合animation规则使用

@font-face

​ @font-face用于定义一种字体，阿里巴巴矢量图标库的icon-font就是利用这个特性实现的

MVC和MVVM的关系

发表于 2022-07-15

​ mvc是服务端的概念，mvvｍ是客户端的概念，他们不能相提并论。

MVC

​ mvc是后端逻辑分层的概念，可以分为m（model），v（view）,c（controller）三大部分。

view

​ view就是视图层的意思，也就是前端页面，视图层 （v）和 控制器（c）通过 http 协议 进行通信，由view请求controler提供的的api接口（request），然后控制器（c）收到请求（request）之后，经过路由（router）将request分发

controler

​ controler就是控制器的意思，通常在基于node.js的接口开发里面，有路由分发模块（router）、 中间件模块（middleware）、控制器模块（controler），这三个模块都属于controler层。从 view 发送到后端的 请求 （request）会先到达入口文件（app.js）,入口文件收到请求之后会调用路由模块（router）进行路由分发,经过路由分发之后,请求会经过中间件(middleware) 进行相应的处理,处理之后会到达controler模块,controler模块会根据请求 调用 数据模型模块（model）,结合model里面的数据并进行逻辑处理之后就会将数据 以http response 的方式返回给 视图层（view）

model

​ model就是数据模型的意思，它是职能单一的，只负责数据库的操作，执行sql语句，进行数据的增、删、改、查 。 数据模型里面对增删改查等操作进行了封装，并对外部暴露了封装好之后的方法，使得controler在对数据库操作的时候不需要每次都写sql语句。同时，model也负责连接数据库。

MVVＭ

​ mｖｖｍ是前端的逻辑分层概念。可以分为三层：每个页面保存的数据模型（ｍ）数据调度者（ｍｖ）视图层（ｖ）。

model

​ 当用户打开一个页面的时候，页面会向服务器发送请求，并将数据保存在本地。这些数据就是当前页面的数据模型。数据模型要显示到页面上的话就要将数据给到ｍｖ。在ｖｕｅ中，响应式数据功能实现了ｍｏｄｅｌ和ｍｏｄｅｌｖｉｅｗ的数据双向绑定

modelview

​ 当数据从服务器下载到本地之后，数据就会流向ｍｖ模型，然后由ｍｖ模型决定什么时候将数据渲染到视图层。当ｍｖ拿到来自于页面的数据之后，也会将数据保存到数据模型中去。在ｖｕｅ中，框架所做的事情就相当于这里的数据调度者。ｖｕｅ框架实现了与ｖｉｅｗ之间的数据双向绑定

view

​ 视图层负责将来自于ｍｖ的数据显示到页面上来。当用户触发页面监听的事件，输入数据之后，将数据返回给数据调度模型（ｍｖ）。

localStorage和sessionstorage的区别

发表于 2022-07-14 更新于 2022-07-19

Web Storage出现的目的是为了解决通过客户端存储解决与服务器通信频繁使用Cookie的问题。

sessionStorage

sessionStorage是跨会话的存储机制，这意味着数据只会存储到浏览器关闭。存储在sessionStorage中的数据不受刷新的影响，可以在浏览器崩溃重启之后恢复。

localStorage

localStorage是永久存储机制，也就是说当浏览器关闭了之后，存储的数据依然在。要访问同一个localStorage对象，页面必须来自同一个域（子域不可以）、相同端口上使用相同的协议。

​ 浏览器给Web Storage使用的空间大小给出了限制，一般会限制每个源5MB。

css选择器和优先级

发表于 2022-07-14

常用的CSS选择器有

  1. 元素选择器 （权重 1）
  2. 群组选择器 （ h2 , p {} 右边花括号里面的规则应用到左边两个选择器上）
  3. 通用选择器 （权重0 * { } ）
  4. 类选择器 （权重10 .class-name { } ）
  5. ID选择器 （权重100 #id { } ）
  6. 后代选择器 （h2 p { } ）
  7. 子元素选择器 （h2 > p { }）
  8. 兄弟选择器 ( h2 + p { } 意思是p元素紧跟在h2元素后面，且同属于一个父元素)
  9. 后续兄弟选择器 ( h2 ~ p { } 意思是同一父元素下所有在h2元素后面的p元素)
  10. 伪类选择器
  11. 伪元素选择器

​ 当有多条规则作用于同一个元素是，渲染引擎就会计算不同规则的权重，由权重最高的那一条规则生效。

css权重的计算就是每出现一个相应权重的选择器，就将它的权值加入总权重当中来。

​ 当元素脱离正常的文档流时，元素之间就会互相堆叠，可以通过设置z-index属性来控制堆叠的顺序。

回流和重绘

发表于 2022-07-14 更新于 2022-07-19

​ 通常浏览器的渲染过程可以分为一下几部分：

  1. HTML被HTML解析器解析成DOM Tree
  2. CSS被CSS解析器解析成CSSOM Tree
  3. DOM Tree 和 CSSOM Tree 解析完成之后合并在一起形成 Render Tree
  4. 渲染绘制

如果将元素设置为display : none 的话，元素就不会被被添加到 Render Tree中，也就是不会被渲染。

回流

​ 当我们通过JavaScript或者CSS改变了元素的宽度、高度、内外边距等，改变了页面的布局，浏览器就会发生回流，相当于刷新页面，对性能消耗比较大。

重绘

​ 当元素的属性发生改变，但是不影响布局的话，比如说颜色，字体样式等等，就会产生重绘，不刷新页面，动态更新内容。对性能消耗相对比回流小。

怎样减少回流和重绘？

​ 尽量使用transform代替位移，使用visibility代替display:none，当使用img标签显示图片的时候，最好给图片同时设置宽高，只设置宽或者高的话会引起重绘。
