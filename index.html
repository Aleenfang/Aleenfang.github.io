微信小程序--分包

发表于 2022-08-03 更新于 2022-08-05

分包

为了加快小程序的加载速度，一般会将小程序分为多个模块，依次加载。

  1. 主包

    1. 主包包含tabBar页面资源，公共资源

  2. 分包

    1. 分包包含非tabBar的页面，私有资源

加载策略

  1. ​ 在小程序启动时，默认下载主包并启动主包里面的页面，也就是tabBar
  2. 当用户进入分包的某个页面的时候，客户端会把对应的分包下载下来，下载完成之后再展示，也就是按需加载

分包的体积限制

  • 整个小程序的所有分包大小不超过16M
  • 单个主包、分包大小不超过2M

使用规则

  • 主包放在主文件夹的pages文件夹里
  • 分包放在主文件加内的分包文件夹里
  • 在app.json里设置的时候 ， 主包 在pages选项里声明，分包在subpackages里面声明

打包规则

  1. 小程序会按subpackages的配置进行分包，subpackages之外的目录将被打包到主包中

  2. 主包也可以有自己的paegs，也就是 app.json里面最外层的pages

  3. tabBar页面必须在主包内

  4. 分包之间不能嵌套，也就是说分包里不能再有分包，但是可以有很多个页面

微信小程序--component

发表于 2022-08-03 更新于 2022-08-05

微信小程序的组件一般放在component文件夹下面

局部引用

当page需要使用到组件的时候，需要在page的json文件中引入component

全局引用

当组件在很多页面上都会用到的时候，就可以在app.json里面对组件进行全局引用。

组件与页面的区别

  • 组件的 .json文件中，要设置 component:”true”
  • 组件的 .js 文件中，调用的时Component() 函数， 页面 调用的时 Page() 函数
  • 组件的事件处理函数，函数等 都 需要定义在methods中

样式隔离

  • app.wxss 中的全局样式对组件无效
  • 只有class选择器有样式隔离效果，id选择器，属性选择器，标签选择器都不受样式隔离的影响
  • 可以在组件的 .json 文件中 或者 .js 文件中 添加 “styleIsolation” 使组件样式隔离失效
    • isolated default：true 表示启用样式隔离
    • apply-shared default: false 表示关闭组件的样式隔离，页面wxss可以影响组件，组件wxss不影响页面
    • shared default :false 表示页面wxss和组件wxss相互影响

组件传值

  • 属性绑定：子组件通过在传入Component函数的对象中添加 properties 属性，来声明要接受的数据名称，这个跟vue 的props类似，这个也可以有完整定义属性的方式和简化定义属性的方式

  • 事件绑定： 跟vue里面类似，子组件给父组件传值可以通过 子组件触发事件，然后 父组件监听子组件触发的事件实现。

    1. 父组件 定义一个函数，用于接受子组件传过来的值 childrenData(e){}
    2. 子组件 通过触发事件将值传给父组件 this.triggerEvent(‘eventName’,{dataObject})
    3. 父组件 通过监听子组件触发的事件， 在接受函数中 拿到 子组件传过来的值 bind:eventName=”childrenData”
    4. 父组件 通过接收函数的参数 获得 子组件传过来的值 e.detail

  • 获取子组件实例：类似于vue里面的组件ref，

    • 父组件通过this.selectComponent(‘id或class选择器’)获取子组件实例，从而直接访问子组件的任意数据和方法，

data 和 properties的区别

在小程序组件中，data和properties都是可读可写的，读取的时候直接用，写入的时候需要使用 setData（{}）函数

  • data 更倾向于存储私有数据
  • properties更倾向于存储外界传递到组件中的数据

纯数据字段

当data中的字段既不会渲染到界面上，也不会传递给其他组件的时候，就可以将它定义为纯数据字段，或者说这个字段不是响应式的。

优点： 有助于提升组件的性能
使用规则：在Component 的 options节点中，指定pureDataPattern为一个正则表达式，符合这个正则表达式的字段将成为纯数据字段。

组件生命周期

  • created 此时还不能调用setData ， 是能用于给this添加一些自定义属性字段
  • attached 这个时候this.data初始化完毕，可以使用setData了
  • ready
  • moved
  • detached 当组件离开页面节点树后触发，退出一个页面会触发所有子组件的这个生命周期，适合做一些清理工作
  • error

组件所在页面的生命周期

在组件内部，可以获取使用该组件的页面的生命周期的情况

  • show 组件所在页面被展示时执行
  • hide 组件所在页面被隐藏时执行
  • resize 组件所在的页面尺寸变化时执行

微信小程序--api三大分类

发表于 2022-08-03 更新于 2022-08-05

微信官方把API分为了如下3大类：

事件监听API

  • 以on开头，类似于vue里面的v-on，但是不能直接给事件处理函数添加参数，需要使用 data-***

同步API

  • 以sync结尾的API都是同步API
  • 同步API的执行结果，可以通过函数返回值直接获取，如果执行异常会抛出异常
  • wx.setStorageSync(‘key’,’value’) 向本地存储中写入数据

异步API

  • 类似于 promise.then() ，catch() , finally() 需要传入success，fail，complete回调函数来处理不同的结果
  • 举个例子： wx.request()发起网络请求，通过success回调函数接受数据，填入fail处理错误
  • 部分API支持promise风格，详情见微信官方wen’dang

微信小程序--tabBar

发表于 2022-08-03 更新于 2022-08-05

导航栏

微信小程序的导航栏设置在 app.json 文件里设置，设置项为 “tabBar”:{ color:’’,selectedColor:’’,list:[] }

  • color 设置导航栏文字未被选中时的颜色

  • selectedColor 设置导航栏被选中时的文字的颜色

  • list 设置导航栏的每个项目的信息

    • “pagePath” 项目的页面地址，点击导航栏会自动跳到这个页面
    • “text” 项目的文字内容
    • “ iconPath” 项目图标的地址
    • “ selectedIconPath” 选中之后的图标的地址
这里面， list里面设置了几个对象，导航栏就会有几个选项

自定义导航栏

  1. 在app.json 里面 的tabBar选项下面设置custom属性为true
  2. 在主文件夹下创建custon-tab-bar目录，并在该目录下创建index组件
  3. 在组件里面写导航栏的相关代码

为了兼容性考虑，需要保存普通的tabBar的设置选项

如果使用vant里面的tabBar组件的话，需要在app.json组件里面注册，不能在custom-tab-bar组件里注册
